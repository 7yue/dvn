DVN SAML/Shibboleth Testing Introduction

Many DVN users are interested in SAML/Shibboleth support: https://redmine.hmdc.harvard.edu/issues/2657

OIOSAML is currently being investigated as an underlying technology as announced at https://lists.iq.harvard.edu/pipermail/dvn-auth/2013-October/000001.html

To begin testing compatibility with Shibboleth Identity Providers (especially from institutions who have people subscribed to the dvn-auth list), we have set up a test page that simply displays attributes from an Identity Provider (IdP) that OIOSAML has been been configured to use. At this time, only one IdP can be tested at at time.

If you run an IdP and are looking for the Service Provider (SP) metadata for this instance of DVN, please download it from here.

If you are interested in seeing the attributes that the IdP is sending (and have a username and password for that IdP), please proceed to the SamlTestPage page.
